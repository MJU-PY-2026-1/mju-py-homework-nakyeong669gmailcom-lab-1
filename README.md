## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
* 이번 과제는 기존 2차 자산 관리 시스템을 발전시켜, 반복문(while True)과 함수(def)를 도입하여 지속적인 관리가 가능한 '메뉴판 시스템'을 구현하는 것입니다. 특히 2차 과제에서 구현했던 입력, 연산, 출력 로직을 개별 함수로 명확히 분리하고, 전역 변수와 지역 변수의 개념을 이해하여 global, return, 매개변수를 적절히 다루는 구조적 프로그래밍을 완성해야 합니다.

* while True 무한 루프를 통해 사용자가 '2번 프로그램 종료'를 누르기 전까지 프로그램이 꺼지지 않고 계속 실행되도록 구현
 
* 기존 기능들을 3개의 개별 함수(input_user_data, process_asset_calculation, output_asset_result)로 나누어 정의
  
* 반환값 활용: input_user_data 함수에서 사용자가 입력한 5개의 값을 return 문을 통해 반환하도록 구현
  
* 매개변수 활용: 연산을 담당하는 process_asset_calculation 함수가 입력값을 인자(매개변수)로 전달받아 처리하도록 구현
  
* global 키워드: 함수 내부에서 프로그램 전체의 자산 데이터(전역 변수)를 직접 업데이트할 수 있도록 global 키워드를 사용
  
* 메인 로직 간결화: while문 내부(메인 로직)에서는 긴 연산 식을 모두 지우고, 목차처럼 순서대로 함수만 호출하도록 간결하게 구성
    
### **🤖 AI 파트너십 과정**
* 내용 1 : 함수 구조화 및 변수 관리 컨셉 도출
프롬프트 요약 : 1학년 파이썬 기초 수준에서 2차 과제 코드를 3개 이상의 함수와 while문 메뉴 시스템으로 바꾸고 싶어. 단, 아직 배우지 않은 복잡한 메인 함수(def main(), if __name__ == '__main__':)나 데이터 입력 체크용 플래그 변수(is_calculated = False)는 쓰지 않고 조건문과 배운 함수 문법만으로 깔끔하게 연동되는 논리적 구조를 제안해 줘.
적용 내용 : AI가 제안한 대로 메뉴를 1번(자산 관리 시작)과 2번(종료)으로 간소화했습니다. 1번을 선택하면 [입력 함수] ➡️ [계산 함수] ➡️ [출력 함수]가 순서대로 강제 실행되도록 설계하여, 아직 배우지 않은 '데이터 입력 여부 체크 변수'를 쓰지 않고도 오류 없이 연산과 출력이 한 번에 이어지도록 코드를 최적화했습니다.

* 내용 2 : 전역 변수(global)와 매개변수의 유기적 연결 자문
프롬프트 요약 : 함수 안에서 변수를 쓰니까 자꾸 값이 초기화되거나 2차 과제 변수들이 유지가 안 돼. 배운 범위 내에서 return과 global을 어떻게 조합해야 기존 자산 관리 변수들을 그대로 유지하면서 함수끼리 데이터를 주고받을 수 있을까
적용 내용 : 입력 함수에서는 받은 데이터를 묶어서 return하고, 이를 메인 루프에서 받아 계산 함수의 매개변수로 던져주는 데이터 전달 흐름을 정립했습니다. 또한, 연산 결과가 일회성으로 사라지지 않고 프로그램 전체에 반영되도록 기존 2차 과제 변수들을 함수 내부에서 global로 선언해 주는 방식을 적용하여 자산 상태를 성공적으로 유지시켰습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**
* 문제 1 : 안 배운 메인 실행 진입점 문법 사용 에러 및 가독성 문제
원인 : 처음 AI 자문을 구했을 때 코드 맨 밑에 if __name__ == "__main__": 구문과 def main(): 함수가 포함되어 있었습니다. 이는 1학년 기초 프로그래밍 주차에서는 배우지 않은 고급 문법이었으며, 제출 시 제한사항(배운 내용만 사용할 것)을 위반하여 감점될 위험이 있었습니다.
해결 : def main():과 진입점 확인 조건문을 과감히 통째로 삭제했습니다. 대신 파이썬의 인터프리터 특징(위에서 아래로 읽는 특징)을 활용해 함수 정의들을 상단에 두고, 프로그램의 중심인 while True 무한 루프를 코드 맨 밑바닥(바깥 영역)에 그대로 노출시켜 배운 내용만으로 직관적으로 실행되도록 수정했습니다.

* 문제 2 : 차 코드의 변수명 오타 발견 및 데이터 불일치
원인 : 2차 과제 코드 분석 중 invest_amount라는 변수명이 조건문 내부 한 곳에서 invest_amonut으로 오타가 나 있는 것을 발견했습니다. 이로 인해 특정 투자 성향 선택 시 변수 값이 정상적으로 대입되지 않고 출력이 0원으로 나오는 논리적 오류(Logic Error)의 가능성이 있었습니다.
해결 : 3차 과제 함수화 작업을 진행하면서 2차 과제 로직 내의 모든 invest_amonut 오타를 정확한 변수명인 invest_amount로 일괄 수정하여, 선택한 투자 성향(안정형/수익형/공격형)에 맞는 비율 계산이 정상적으로 누락 없이 출력 리포트에 반영되도록 해결했습니다.
