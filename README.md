# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
시나리오 제목 :  대학생 맞춤형 월간 수입 및 투자 가용 자산 계산 프로그램



## 2. 시나리오 (5~10줄)
이 프로그램은 대학생의 불규칙한 수입과 고정 지출을 분석하여 효율적인 자산 배분을 도와주
는 개인 맞춤형 재무 관리 시스템입니다.
사용자는 매달 변동되는 아르바이트비와 공과금(전기, 가스) 정보를 입력합니다. 시스템은 미
리 설정된 고정 지출 항목(생활비, 구독료, 청약 등)과 합산하여 총 수입 대비 지출 비중을 계
산합니다.
특히, 남은 잔액을 주식 투자 70%와 비상금 저축 30%의 비율로 자동 배분하여 제안합니다. 
또한, 매달 납입하는 주택청약의 누적 횟수와 총액을 자동으로 업데이트하여 자산 형성 과정
을 시각화합니다.
최종적으로는 메뉴판 기능을 통해 월별 결산, 투자 가이드 조회, 청약 현황 관리 등 다각도로 
자산을 관리할 수 있는 시스템을 완성할 계획입니다.

## 3. 예상 기능 및 메뉴 (최소 5개)
1.이번 달 수입 및 변동 지출 입력
2.월간 고정 지출 및 총지출 분석
3.투자 가능 금액 및 비상금 배분 계산
4.주택청약 납입 현황(횟수 및 잔액) 업데이트
5.월간 재무 요약 리포트 출력
6.프로그램 종료

## 🟦 [1차 과제: V1.0] 시나리오 기획
V1.0 개발 내용: 사용자로부터 이번 달 알바비, 전기세, 가스비, 현재 청약 상태를 입력받아 
최종 가용 자산을 계산하여 출력합니다. 수입/지출(정수형), 투자 비율(실수형), 청약 횟수(정
수형) 등 다양한 자료형을 활용하여 데이터를 구조화했습니다. 사칙연산을 활용하여 잔액 배
분(7:3) 로직과 청약 누계 계산을 구현했습니다.    

### **🤖 AI 파트너십 과정**
1. 내용 
프롬프트 요약: 입출력 → 조건문 → 반복문 → 함수/배열로 확장 가능한 대학생 자산 관리 
프로그램 시나리오 기획 요청
적용 내용: 단순 계산기를 넘어, 기초 입력(1차) → 지출 과다 판별(2차) → 메인 메뉴 시스템
(3차) → 월별 데이터 관리(4차)로 이어지는 단계별 확장 로드맵을 확립함.
2. 내용 
프롬프트 요약: 실제 생활비 데이터를 기반으로 한 변수 설계 및 투자/비상금 배분 로직 논의
적용 내용: 고정 지출(생활비, OTT 등)은 상수로 관리하고, 변동 지출만 입력받는 구조를 설
계함. 특히 투자금 계산 시 실수형(float) 자료형을 사용하여 70% 비율 계산의 정확도를 높임.

￿ Troubleshooting & 기술 회고:
문제 1: 사용자에게 알바비를 입력받을 때 숫자가 아닌 문자가 섞일 경우 오류가 발생할 가능
성 확인.
해결: 현재 단계에서는 정수형 입력을 가정하되, int(input())을 사용하여 입력값이 산술 연산
이 가능하도록 명확히 형변환함.
문제 2: 투자금 계산 결과(* 0.7)가 소수점 아래로 너무 길게 출력되어 가독성이 떨어짐.
해결: 파이썬의 f-string 포맷(:.0f)을 사용하여 소수점 없이 반올림된 정수 형태로 출력하여 
깔끔한 리포트를 구현함.
문제 3: 수입보다 지출이 큰 경우 마이너스 잔액이 발생함.
해결: 1차 과제에서는 조건문 사용이 금지되어 있으므로, 현재는 그대로 출력하되 2단계 과제
에서 if문을 활용해 '잔액 부족 경고' 기능을 추가하기로 계획함

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
