# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
시나리오 제목 :  대학생 맞춤형 월간 수입 및 투자 가용 자산 계산 프로그램



## 2. 시나리오 (5~10줄)
이 프로그램은 대학생의 불규칙한 수입과 고정 지출을 분석하여 효율적인 자산 배분을 도와주
는 개인 맞춤형 재무 관리 시스템입니다.
사용자는 매달 변동되는 아르바이트비와 공과금(전기, 가스) 정보를 입력합니다. 시스템은 미
리 설정된 고정 지출 항목(생활비, 구독료, 청약 등)과 합산하여 총 수입 대비 지출 비중을 계
산합니다.
특히, 남은 잔액을 주식 투자 70%와 비상금 저축 30%의 비율로 자동 배분하여 제안합니다. 
또한, 매달 납입하는 주택청약의 누적 횟수와 총액을 자동으로 업데이트하여 자산 형성 과정
을 시각화합니다.
최종적으로는 메뉴판 기능을 통해 월별 결산, 투자 가이드 조회, 청약 현황 관리 등 다각도로 
자산을 관리할 수 있는 시스템을 완성할 계획입니다.

## 3. 예상 기능 및 메뉴 (최소 5개)
1.이번 달 수입 및 변동 지출 입력
2.월간 고정 지출 및 총지출 분석
3.투자 가능 금액 및 비상금 배분 계산
4.주택청약 납입 현황(횟수 및 잔액) 업데이트
5.월간 재무 요약 리포트 출력
6.프로그램 종료

## 🟦 [1차 과제: V1.0] 시나리오 기획
V1.0 개발 내용: 사용자로부터 이번 달 알바비, 전기세, 가스비, 현재 청약 상태를 입력받아 
최종 가용 자산을 계산하여 출력합니다. 수입/지출(정수형), 투자 비율(실수형), 청약 횟수(정
수형) 등 다양한 자료형을 활용하여 데이터를 구조화했습니다. 사칙연산을 활용하여 잔액 배
분(7:3) 로직과 청약 누계 계산을 구현했습니다.    

### **🤖 AI 파트너십 과정**
1. 내용 
프롬프트 요약: 입출력 → 조건문 → 반복문 → 함수/배열로 확장 가능한 대학생 자산 관리 
프로그램 시나리오 기획 요청
적용 내용: 단순 계산기를 넘어, 기초 입력(1차) → 지출 과다 판별(2차) → 메인 메뉴 시스템
(3차) → 월별 데이터 관리(4차)로 이어지는 단계별 확장 로드맵을 확립함.
2. 내용 
프롬프트 요약: 실제 생활비 데이터를 기반으로 한 변수 설계 및 투자/비상금 배분 로직 논의
적용 내용: 고정 지출(생활비, OTT 등)은 상수로 관리하고, 변동 지출만 입력받는 구조를 설
계함. 특히 투자금 계산 시 실수형(float) 자료형을 사용하여 70% 비율 계산의 정확도를 높임.

￿ Troubleshooting & 기술 회고:
문제 1: 사용자에게 알바비를 입력받을 때 숫자가 아닌 문자가 섞일 경우 오류가 발생할 가능
성 확인.
해결: 현재 단계에서는 정수형 입력을 가정하되, int(input())을 사용하여 입력값이 산술 연산
이 가능하도록 명확히 형변환함.
문제 2: 투자금 계산 결과(* 0.7)가 소수점 아래로 너무 길게 출력되어 가독성이 떨어짐.
해결: 파이썬의 f-string 포맷(:.0f)을 사용하여 소수점 없이 반올림된 정수 형태로 출력하여 
깔끔한 리포트를 구현함.
문제 3: 수입보다 지출이 큰 경우 마이너스 잔액이 발생함.
해결: 1차 과제에서는 조건문 사용이 금지되어 있으므로, 현재는 그대로 출력하되 2단계 과제
에서 if문을 활용해 '잔액 부족 경고' 기능을 추가하기로 계획함

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
* 이번 과제는 ‘대학생 맞춤형 자산 관리 매니저’ 프로그램으로, 사용자의 불규칙한 수입(알바비)과 변동 지출(공과금)을 입력받아 재무 상태를 자동으로 진단하는 시스템을 구축하는 것입니다. 특히 if-elif-else를 활용한 다중 조건 구조와 if문 내부에 if문을 넣는 중첩 구조, 그리고 and 논리 연산자를 활용하여 투자가 가능한 ‘최적의 상황’을 판별하는 로직을 완성했습니다.

* input()과 int() 형변환을 통해 알바비, 전기세, 가스비를 입력받음.

* if-elif-else를 사용하여 적자 상태, 주의 상태, 여유 상태의 3단계 재무 등급 판정 로직 구현.

* 투자 메뉴 선택 기능을 구현하여 성향에 맞는 투자 배분 로직을 추가하고, and 연산자를 활용하여 ‘잔액이 충분할 때만’ 투자가 실행되도록 안전장치 구현.


### **🤖 AI 파트너십 과정**
* 내용 1: 파이썬식 ‘Switch-Case’ 구현 및 로직 설계
프롬프트 요약: "파이썬에는 switch-case 문법이 없는데, 교수님께서 요구하신 단일값 판별 메뉴 기능을 어떻게 구현해야 할까? 그리고 중첩 조건문을 효과적으로 넣는 방법이 궁금해."
적용 내용: 파이썬의 if-elif-else 구조를 활용하여 switch-case와 동일한 기능을 수행하도록 설계함. 또한, 흑자 상태일 때만 투자 메뉴를 띄우도록 else 문 내부에 투자 선택 로직을 넣는 '중첩 구조'를 적용함.

*내용 2: 논리 연산자를 활용한 자산 관리 안전장치
프롬프트 요약: "잔액이 부족한데도 무리하게 투자가 진행되는 것을 막고 싶어. and 연산자를 써서 '잔액이 10만 원 이상일 때만 투자 선택지가 작동하게' 해줘."
적용 내용: if balance >= 100000 and invest_choice == 1:과 같이 복합 조건식을 사용하여, 경제적 상황과 사용자의 선택을 동시에 체크하는 안전한 투자 로직을 완성함.
    
### **🛠️ Troubleshooting & 기술 회고:**
* 문제 1: 적자 발생 시 프로그램의 무분별한 계산
원인: 1차 과제에서는 지출이 수입보다 커도 무조건 투자금이 계산되어 마이너스 값이 나오는 논리적 오류가 있었음.
해결: if-elif-else 다중 조건문을 사용하여, 적자일 경우 '경고' 메시지를 출력하고 프로그램의 흐름을 안전하게 차단하는 로직으로 수정함.

* 문제 2: 파이썬에서의 switch-case 문법 부재로 인한 고민
원인: 교수님의 요구사항에 switch-case 사용이 명시되어 있었으나, 파이썬 기초 단계에서 파이썬에 없는 문법을 억지로 쓰려다 혼란이 발생함.
해결: 파이썬에서는 if-elif를 사용하는 것이 표준이자 올바른 접근임을 학습하고, 이를 통해 교수님의 의도(단일값 판별 및 분기)를 완벽하게 구현함.

* 문제 3: 중첩 조건문 사용 시 가독성 저하
원인: if문 안에 if문을 계속 넣으니 코드가 오른쪽으로 길어져서 읽기 힘들어짐.
해결: 조건식의 순서를 조정하여 가장 중요한 조건(적자 여부)을 먼저 체크하고, 통과한 경우에만 중첩 조건문으로 진입하게 하여 코드의 가독성을 높임.
